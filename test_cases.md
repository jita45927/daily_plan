# 每日计划应用测试用例清单

## 一、窗口基础特性测试

### TC-WIN-001：窗口置顶功能
- **测试目的**：验证窗口置顶功能是否正常工作
- **前置条件**：应用已启动，窗口处于正常显示状态
- **测试步骤**：
  1. 点击窗口置顶按钮/菜单
  2. 打开其他应用窗口（如浏览器、资源管理器）
  3. 观察每日计划窗口是否保持在最上层
  4. 再次点击置顶按钮取消置顶
  5. 打开其他应用窗口，观察窗口是否不再置顶
- **预期结果**：
  - 置顶开启后，窗口始终显示在其他窗口之上
  - 置顶关闭后，窗口可被其他窗口覆盖

### TC-WIN-002：窗口拖拽功能
- **测试目的**：验证窗口拖拽移动功能
- **前置条件**：应用已启动，窗口未被锁定
- **测试步骤**：
  1. 鼠标按住窗口标题栏/拖拽区域
  2. 拖动窗口到屏幕其他位置
  3. 释放鼠标，观察窗口位置
- **预期结果**：
  - 窗口跟随鼠标移动
  - 释放鼠标后，窗口停留在新位置

### TC-WIN-003：窗口贴边隐藏-顶部
- **测试目的**：验证窗口拖动到屏幕顶部时自动隐藏
- **前置条件**：应用已启动，窗口未被锁定
- **测试步骤**：
  1. 将窗口拖动到屏幕顶部边缘（距离顶部约10px以内）
  2. 观察窗口变化
  3. 将鼠标移动到顶部边缘，观察窗口是否展开
  4. 再次拖动窗口离开顶部，观察窗口是否恢复正常大小
- **预期结果**：
  - 窗口拖动到顶部时自动收缩为窄条（约8px高度）
  - 鼠标悬停顶部边缘时窗口展开
  - 拖动离开顶部后恢复原始大小和位置

### TC-WIN-004：窗口贴边隐藏-左侧
- **测试目的**：验证窗口拖动到屏幕左侧时自动隐藏

### TC-WIN-008：窗口宽度固定
- **测试目的**：验证窗口宽度固定为400px，无法调整
- **前置条件**：应用已启动
- **测试步骤**：
  1. 尝试拖动窗口左右边缘调整宽度

## 三、桌面文件整理测试

### TC-DSK-001：文件夹创建
- **测试目的**：验证整理桌面时自动创建目标文件夹
- **前置条件**：桌面上不存在"快捷方式"和"其他文件"文件夹
- **测试步骤**：
  1. 点击桌面整理按钮
  2. 检查桌面是否创建了"快捷方式"和"其他文件"文件夹
- **预期结果**：
  - 成功创建"快捷方式"文件夹
  - 成功创建"其他文件"文件夹

### TC-DSK-002：快捷方式分类
- **测试目的**：验证.lnk快捷方式文件被移动到"快捷方式"文件夹
- **前置条件**：桌面上存在多个.lnk快捷方式文件
- **测试步骤**：
  1. 在桌面上放置1-2个快捷方式文件（.lnk）
  2. 点击桌面整理按钮，选择任意冲突策略
  3. 检查"快捷方式"文件夹内容
  4. 检查桌面原始位置
- **预期结果**：
  - 所有.lnk文件被移动到"快捷方式"文件夹
  - 桌面不再包含这些.lnk文件

### TC-DSK-003：其他文件分类
- **测试目的**：验证非快捷方式文件和文件夹被移动到"其他文件"文件夹
- **前置条件**：桌面上存在非.lnk文件和文件夹
- **测试步骤**：
  1. 在桌面上放置文档、图片等文件和文件夹
  2. 点击桌面整理按钮，选择任意冲突策略
  3. 检查"其他文件"文件夹内容
  4. 检查桌面原始位置
- **预期结果**：
  - 所有非.lnk文件被移动到"其他文件"文件夹
  - 所有文件夹被移动到"其他文件"文件夹
  - 桌面不再包含这些文件和文件夹

### TC-DSK-004：同名冲突处理-覆盖策略
- **测试目的**：验证同名文件冲突时覆盖策略生效
- **前置条件**："快捷方式"或"其他文件"文件夹中已存在同名文件
- **测试步骤**：
  1. 在目标文件夹中创建文件A.txt
  2. 在桌面上创建内容不同的同名文件A.txt
  3. 点击桌面整理按钮，选择覆盖策略
  4. 检查目标文件夹中文件A.txt的内容
- **预期结果**：
  - 目标文件夹中的文件被桌面上的文件覆盖
  - 文件内容为桌面文件的内容

### TC-DSK-005：同名冲突处理-重命名策略
- **测试目的**：验证同名文件冲突时重命名策略生效
- **前置条件**："快捷方式"或"其他文件"文件夹中已存在同名文件
- **测试步骤**：
  1. 在目标文件夹中创建文件A.txt
  2. 在桌面上创建内容不同的同名文件A.txt
  3. 点击桌面整理按钮，选择重命名策略

- **预期结果**：
  - 桌面上的文件被重命名为A(1).txt并移动到目标文件夹
  - 原文件A.txt保留不变

### TC-DSK-006：同名冲突处理-跳过策略
- **测试目的**：验证同名文件冲突时跳过策略生效
- **前置条件**："快捷方式"或"其他文件"文件夹中已存在同名文件

  2. 在桌面上创建内容不同的同名文件A.txt

  - 桌面上的文件保持不动（未被移动）

### TC-DSK-007：异常处理-桌面路径不存在
- **测试目的**：验证桌面路径不存在时的错误处理
- **前置条件**：模拟桌面路径不存在的情况

## 五、限时/定时倒计时测试

### TC-TIMER-001：限时倒计时-正常启动
- **测试目的**：验证限时倒计时功能正常启动
- **前置条件**：任务列表中存在任务
- **测试步骤**：
  1. 选择一个任务
  2. 设置限时倒计时（如5分钟）
  3. 点击启动按钮
  4. 观察倒计时显示
- **预期结果**：
  - 倒计时开始倒计时
  - 显示格式为 HH:MM:SS
  - 每秒更新一次

### TC-TIMER-002：限时倒计时-时间范围验证
- **测试目的**：验证倒计时时间范围限制（1-1440分钟）
- **前置条件**：任务列表中存在任务
- **测试步骤**：
  1. 尝试设置0分钟倒计时
  2. 观察错误提示
  3. 尝试设置1441分钟倒计时
  4. 观察错误提示
  5. 设置1分钟和1440分钟倒计时，验证正常启动
- **预期结果**：
  - 0分钟和1441分钟返回错误信息
  - 1分钟和1440分钟正常启动

### TC-TIMER-003：限时倒计时-结束通知
- **测试目的**：验证倒计时结束时发送通知
- **前置条件**：任务列表中存在任务
- **测试步骤**：
  1. 设置1分钟倒计时
  2. 启动倒计时
  3. 等待倒计时结束
  4. 观察通知/弹窗
- **预期结果**：
  - 倒计时归零时触发通知
  - 收到"timer_expired"事件

### TC-TIMER-004：定时倒计时-正常启动
- **测试目的**：验证定时倒计时功能正常启动
- **前置条件**：任务列表中存在任务
- **测试步骤**：
  1. 选择一个任务
  2. 设置目标时间戳（当前时间+5分钟）
  3. 点击启动按钮
  4. 观察倒计时显示
- **预期结果**：
  - 倒计时开始倒计时
  - 显示格式为 HH:MM:SS
  - 每秒更新一次

### TC-TIMER-005：定时倒计时-时间验证
- **测试目的**：验证目标时间必须大于当前时间
- **前置条件**：任务列表中存在任务
- **测试步骤**：
  1. 设置目标时间戳为当前时间之前
  2. 点击启动按钮
  3. 观察错误提示
- **预期结果**：
  - 返回错误信息"目标时间必须大于当前时间"
  - 倒计时不启动

### TC-TIMER-006：停止倒计时
- **测试目的**：验证停止倒计时功能
- **前置条件**：倒计时正在运行中

### TC-TIMER-007：获取倒计时状态
- **测试目的**：验证获取倒计时状态功能
- **前置条件**：倒计时正在运行中

  3. 观察返回结果
- **预期结果**：
  - 返回任务ID、剩余时间、格式化时间、运行状态等信息

### TC-TIMER-008：倒计时校准
- **测试目的**：验证倒计时校准功能
- **前置条件**：倒计时正在运行中
